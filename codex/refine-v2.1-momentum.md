# v2.1 보완 — 모멘텀 칩 정련 (이전 세션 이어서)

직전 작업으로 카드에 두 모멘텀 칩(주간 변화 `─`, 점수정체 `정체 Nd`)을 넣었다. 문제는 `정체 Nd`가 진척 점수(total) 변화만 보고 판단해서, **매일 커밋하는 최활성 프로젝트(예: Formi, 커밋 215, lastUpdate 2026-05-26)에도 "정체 12d"가 붙어** 오해를 준다. 점수 정체와 실제 활동 중단은 다르다. 게다가 suggestions 기반 블로커 뱃지(`⚠ 정체 …`)와 단어가 겹쳐 혼동된다.

## 수정 (dashboard.html만)

`renderMasterBoard()`와 `renderTable()`의 모멘텀 칩 2번째를 **점수정체 → 커밋 최신성**으로 바꾼다.

1. 첫 칩(주간 점수 변화)은 유지하되 `─`를 `보합`으로 표기(초록 `▲ +N` / 빨강 `▼ −N` / 회색 `보합`).
2. 둘째 칩을 `정체 Nd`(점수기반) 대신 **마지막 커밋 경과일**로 교체한다.
   - 기준일 `META.asOf` 와 프로젝트 `last`(lastUpdate, yyyy-mm-dd)의 차이 일수 D.
   - 표기 `커밋 D일 전`(D=0이면 `오늘 커밋`, 1이면 `어제 커밋`).
   - 색: D ≤ 3 초록 계열, 4 ≤ D ≤ 13 회색(중립), D ≥ 14 앰버(휴면 신호). "정체"라는 단어는 이 칩에서 쓰지 말 것 — 블로커 뱃지와 겹치니까.
3. 블로커 뱃지(suggestions high severity)는 그대로 둔다. 그것이 진짜 "정체/위험" 신호다.
