완성도 측정 기준

완성도 = 문서(0-20) + 골격(0-30) + 기능(0-30) + 알파(0-20) = 0-100. 활동량은 이 점수를 자동으로 올리지 않고, 모멘텀·건강·신뢰도에서 따로 봅니다.

문서 (0-20) — PRD·스펙·ADR 문서화
  • 0-4 README 한두 줄
  • 5-9 README에 컨셉+실행법+구조
  • 10-15 PRD 또는 Spec 1개 + 컨셉 문서
  • 16-20 PRD+ADR+로드맵+운영규약 등 다층 문서
골격 (0-30) — 하네스 엔지니어링 + 서브에이전트 구성
  • 0-6 실행 골격만, 하네스·에이전트 구성 없음
  • 7-14 빌드 골격+디렉토리 구조, 하네스 초기
  • 15-22 검증 하네스(check.sh/CI) 동작 + 서브에이전트 역할 분담
  • 23-30 하네스+서브에이전트+자동 리뷰/플레이테스트 프로토콜 정착
기능 (0-30) — PRD 핵심 기능 동작 비율
  • 0-5 거의 미구현, 랜딩/데모만
  • 6-12 핵심 1-2개 mock 동작
  • 13-19 PRD 절반 실데이터 동작
  • 20-25 PRD 80% 동작
  • 26-30 PRD 100% + 폴리시
알파 (0-20) — 검증·사용자 테스트·폴리시
  • 0-2 내부 검증 미실시
  • 3-7 합성 플레이테스트 또는 셀프 QA
  • 8-13 알파 사용자 N=5+ 보고서
  • 14-18 알파 N=10+ 종합 보고서
  • 19-20 클로즈드 베타 진입 가능
운영 신호 — 점수와 분리해서 보는 항목
  • 모멘텀은 최근 커밋과 7일 완성도 변화를 봅니다
  • 건강은 정체·블로커·스프린트 위험을 봅니다
  • 신뢰도는 문서·히스토리·다음 액션 등 증거의 밀도를 봅니다
