# /weekly-report — 주간 진척 리포트

`projects.json`(과 있으면 `suggestions.json`)을 읽어 한 주의 진척 상황을 요약한 리포트를 만든다. 사용자가 주간 회고·계획을 빠르게 훑을 수 있게 한다.

인자로 날짜나 기간이 주어지면(`$ARGUMENTS`) 그 기준으로, 없으면 `meta.asOf` 기준 직전 7일로 본다.

## 리포트 구성

마크다운으로 아래 순서를 산출한다.

1. **헤더** — 리포트 기간, `meta.asOf`, 활성/일시중단 개수
2. **이번 주 하이라이트** — 진척도·커밋이 가장 많이 움직인 프로젝트 2-3개
3. **Sprint 현황** — Sprint A~D별로 속한 프로젝트와 진척도. 마감 임박 Sprint를 먼저
4. **프로젝트별 한 줄** — 활성 프로젝트 각각: `이름 — 진척 N% · 커밋 N · 막힌 단계 · 다음 액션`
5. **주의 신호** — 정체(`lastUpdate` 4일+) 또는 `/coach`가 `high`로 표시한 항목
6. **다음 주 우선 착수** — `rank` 상위 + Sprint 임박 기준 2-3개, 각 구체적 다음 액션

## 규칙

- `projects.json`의 수치를 그대로 쓴다. `progress` 점수를 임의로 바꾸지 않는다.
- 분석이 필요하면 `/coach`의 판단 기준(막힌 단계·속도 추세·Sprint 적합성·에스컬레이션)을 재사용한다.
- 데이터가 오래됐으면(`meta.asOf`가 며칠 전) 먼저 `/refresh`를 권한다.

## 마무리

- 리포트를 채팅으로 사용자에게 보여준다.
- 사용자가 원하면 `docs/weekly-report-YYYY-MM-DD.md`로 저장한다 (먼저 저장 여부를 묻는다).
