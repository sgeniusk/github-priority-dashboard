# Loom · 유튜브 운영 시스템 — PRD

> 2026-07-21 Loom 운영 핸드오프와 README를 기준으로 만든 대시보드 미러입니다. 정식 `/sync-project pliside-ops` 전까지 `project.json`의 `synced`는 false로 유지합니다.

## 개요

도돌이·팜브리즈·마루 음악 채널을 한 엔진으로 운영하는 로컬 OS입니다. SQLite를 단일 진실원으로 삼아 곡, 앨범, 영상, 쇼츠, 메타데이터, 예약, 분석과 공개 패키지를 관리합니다. 음악·이미지 생성과 최종 업로드는 사람 승인 경계로 유지합니다.

## 기술 스택

- Python 3.12.
- Typer와 Pydantic.
- SQLite.
- Next.js.
- Remotion과 ffmpeg.

## 핵심 기능

- 채널·아티스트·앨범·트랙·영상의 운영 원장.
- 프롬프트, 음원 분석, 마스터링, 영상과 쇼츠 제작 패키지.
- YouTube 메타데이터·현지화·예약·동기화 보조.
- 채널별 공개 캘린더와 성과 분석.
- 모든 쓰기 명령의 dry-run과 운영자 승인 경계.

## 리스크

- Suno·이미지·YouTube 업로드가 운영자 승인과 수동 작업에 의존합니다.
- 여러 채널의 예약·DB·공개 상태가 어긋날 수 있습니다.
- 대용량 음원·영상 자산과 렌더 저장공간 관리가 필요합니다.
- 지속 운영형이라 완성 백분율을 계산하지 않습니다.
